# Infrastructure Design

## Purpose
Map logical components to actual cloud services and define deployment architecture.

## Prerequisites
- Functional Design complete
- NFR Design complete (if executed)

## Step 1: Analyze Design Artifacts
- Load the Functional Design and NFR Design Google Drive files for this unit
- Identify components needing infrastructure mapping

## Step 2: Generate Infrastructure Artifacts

Create an **Infrastructure Design** Google Drive file under Design → {unit-name} with the following sections:

- **Infrastructure Design** — Service mapping (compute, storage, messaging, networking)
- **Deployment Architecture** — Deployment model, scaling, environments

Must comply with applicable Ancestry standards (ENGP-911, ENGP-849, ENGP-895, ENGP-877).

## Step 3: Present for Approval

```markdown
# 🏢 Infrastructure Design Complete - [unit-name]

> **📋 REVIEW REQUIRED:**
> Please examine the Infrastructure Design in Google Drive under Design → [unit-name] → Infrastructure Design
>
> **You may:**
> 🔧 **Request Changes** — Modify the infrastructure design
> ✅ **Continue to Next Stage** — Proceed to **Code Generation**
```

## Step 4: Log and Update State
- Log approval in the `audit` Google Drive file
- Mark Infrastructure Design complete in the `aidlc-state` Google Drive file
