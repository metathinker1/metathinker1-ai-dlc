# NFR Design

## Purpose
Determine non-functional requirements and design patterns for the unit — performance, security, scalability, tech stack, resilience.

## Prerequisites
- Functional Design complete for the unit

## Step 1: Analyze Functional Design
- Load the Functional Design Google Drive file for this unit
- Identify NFR concerns (performance, security, availability, scalability)

## Step 2: Generate NFR Artifacts

Create an **NFR Design** Google Drive file under Design → {unit-name} with the following sections:

- **NFR Requirements** — Scalability, performance, availability, security requirements
- **Tech Stack Decisions** — Technology choices and rationale
- **Design Patterns** — Resilience patterns, caching strategy, scaling mechanisms, security implementation

## Step 3: Present for Approval

```markdown
# 🎨 NFR Design Complete - [unit-name]

> **📋 REVIEW REQUIRED:**
> Please examine the NFR Design in Google Drive under Design → [unit-name] → NFR Design
>
> **You may:**
> 🔧 **Request Changes** — Modify the NFR design
> ✅ **Continue to Next Stage** — Proceed to next stage
```

## Step 4: Log and Update State
- Log approval in the `audit` Google Drive file
- Mark NFR Design complete in the `aidlc-state` Google Drive file
